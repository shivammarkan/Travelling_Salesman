TSP Explorer

Exploring the complexities of combinatorial optimisation with the Travelling Salesman Problem.

What is TSP?

The Travelling Salesman Problem asks: "Given a list of cities and the distances between each pair, what is the shortest possible route that visits each city exactly once and returns to the origin city?"

NP-Hard Search

TSP is NP-hard. This means that as you add more cities, the time needed to find the absolute shortest path grows factorially ($O(n!)$). For just 20 cities, there are $2.43 \times 10^{18}$ possible routes!

Algorithms

Nearest Neighbour: A "greedy" approach that always picks the closest unvisited city. It's fast but often sub-optimal.

2-Opt: A local search algorithm that iteratively improves a path by swapping edges to remove intersections.

Real-world Use

TSP logic is used in logistics for delivery routes, DNA sequencing, and even in manufacturing circuit boards.
